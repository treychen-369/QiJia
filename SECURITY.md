## 报告安全漏洞

如果您发现了安全漏洞，**请不要**通过公开的 GitHub Issue 报告。

### 负责任的披露

请通过以下方式之一联系我们：

1. **GitHub Security Advisories** — 在仓库的 Security 标签页创建私密安全报告
2. **Email** — 发送邮件至项目维护者（见 GitHub 个人主页）

### 报告内容

请在报告中包含：

- 漏洞描述
- 复现步骤
- 影响范围
- 建议的修复方案（如有）

### 响应时间

- **确认收到**：48 小时内
- **初步评估**：5 个工作日内
- **修复发布**：根据严重程度，通常 7-30 天

### 安全最佳实践

部署 QiJia 时，请确保：

- [ ] 使用强密码作为 `NEXTAUTH_SECRET`（至少 32 字符）
- [ ] 使用强密码作为 `POSTGRES_PASSWORD`
- [ ] 不要将 `.env.local` 或 `.env.prod` 提交到代码仓库
- [ ] 生产环境启用 HTTPS
- [ ] 定期更新依赖包
- [ ] 限制数据库访问权限

感谢你帮助保护 QiJia 的安全！
