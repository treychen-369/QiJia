# 故障排查指南

## 问题：加载数据失败 - 获取仪表板数据失败

### 症状
访问仪表板页面时显示错误："加载数据失败: 获取仪表板数据失败" 或 "未授权访问"

### 原因
这个错误通常是因为：
1. **用户未登录**: 最常见的原因，需要先登录才能访问仪表板
2. **会话过期**: 登录会话已过期，需要重新登录
3. **认证配置问题**: NextAuth配置可能有问题

### 解决方案

#### 1. 登录系统（推荐）
1. 访问首页: http://localhost:3000
2. 如果没有账户，先注册
3. 使用有效的邮箱和密码登录
4. 登录成功后会自动跳转到仪表板

#### 2. 检查现有用户
系统中已有的测试用户：
- Email: test@example.com
- Email: user@example.com

如果忘记密码，需要通过数据库重置或重新注册。

#### 3. 验证认证状态
打开浏览器开发者工具 (F12)，在Console中运行：
```javascript
// 检查NextAuth会话
fetch('/api/auth/session')
  .then(r => r.json())
  .then(data => console.log('会话状态:', data))
```

如果返回空对象 `{}`，说明未登录。

#### 4. 清除浏览器缓存（如果认证异常）
1. 打开浏览器设置
2. 清除所有cookies和缓存
3. 重新访问网站并登录

## 问题：API返回401未授权

### 解决步骤
1. **确认已登录**: 检查是否显示用户名和头像
2. **重新登录**: 退出后重新登录
3. **检查cookies**: 确保浏览器允许cookies
4. **检查CORS设置**: 如果使用不同域名访问，检查CORS配置

## 问题：数据显示不正确

### 可能的原因
1. **数据库数据问题**: 账户余额或持仓数据可能不完整
2. **计算逻辑错误**: 后端计算逻辑可能有bug
3. **类型转换问题**: 数值类型转换错误

### 解决步骤
1. **检查数据库数据**:
   ```bash
   npm run db:studio
   ```
   查看 AccountBalance 和 Holding 表的数据

2. **重新导入数据**:
   如果数据有问题，可以：
   - 删除现有数据
   - 重新导入Excel文件

3. **查看服务器日志**:
   检查终端中的错误信息

## 问题：前端显示混乱

### 症状
- 数字不对
- 图表不显示
- 样式错乱

### 解决步骤
1. **刷新页面**: 按 Ctrl+R 或 F5
2. **硬刷新**: 按 Ctrl+Shift+R 清除缓存刷新
3. **检查浏览器兼容性**: 建议使用Chrome或Edge最新版本
4. **查看浏览器控制台**: 按F12查看是否有JavaScript错误

## 调试工具

### 1. 查看服务器日志
开发服务器终端会显示所有请求和错误信息。

### 2. 使用Prisma Studio
```bash
npm run db:studio
```
可视化查看和编辑数据库数据。

### 3. 测试API
使用curl测试API是否正常：
```bash
# 测试仪表板API（需要登录）
curl http://localhost:3000/api/dashboard

# 如果返回 {"error":"未授权访问"}，说明API正常，只是需要登录
```

### 4. 检查数据库连接
```bash
node -e "const { PrismaClient } = require('@prisma/client'); const prisma = new PrismaClient(); prisma.user.findMany().then(users => { console.log('找到', users.length, '个用户'); prisma.$disconnect(); });"
```

## 常见错误码

| 错误码 | 含义 | 解决方案 |
|-------|------|---------|
| 401 | 未授权 | 需要登录 |
| 403 | 禁止访问 | 权限不足 |
| 404 | 未找到 | 路径错误或数据不存在 |
| 500 | 服务器错误 | 查看服务器日志 |

## 获取帮助

如果以上方法都无法解决问题：
1. 检查服务器终端的详细错误信息
2. 查看浏览器开发者工具的Console和Network标签
3. 确保数据库正常运行
4. 确保所有依赖已正确安装 (`npm install`)

## 快速重置

如果问题持续存在，可以尝试完全重置：

```bash
# 1. 停止服务器 (Ctrl+C)

# 2. 重置数据库
npm run db:reset

# 3. 重新启动
npm run dev

# 4. 重新注册用户并导入数据
```

**注意**: 重置数据库会删除所有数据！
